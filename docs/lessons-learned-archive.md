| L011 | 2026-05-02 | 5c + 5c bis (2026-05-03) | @fullstack (post-arbitrage Thomas) | P2 | **Pattern « filtre amplitude minimum = absorption frais » sur produits dérivés à frais fixes** : sur tout instrument à frais fixes non négligeables (turbos BD 1.98 € + spread 0.05 € + slippage 0.1 %), un signal n'a de valeur économique que si le mouvement attendu sur **sous-jacent** dépasse un seuil minimum d'absorption (ici 1 %). Implémenter ce filtre à 2 niveaux : (1) **amont** dans le générateur de signal (backtester) — évite de produire des candidats marginaux ; (2) **aval** dans les sanity checks LLM-side (SC2 enrichi) — défense en profondeur si l'amont a une dérive. Filtre amont = `min_amplitude_pct=0.01` paramètre par défaut H-A/H-C. Filtre aval = SC2 v1.4 (NO_TRADE forcé < 0.5 %, plafond 6.0 < 1 %). Réutilisable sur tout produit dérivé/option/turbo où frais fixes existent et où la « probabilité de gain × amplitude attendue » doit dépasser un breakeven net. **v1.4 (Phase 5c bis, 2026-05-03)** : amplitude STRICTEMENT calculée sur **prix sous-jacent** (`underlying_entry`/`underlying_tp` dans context), PAS sur prix turbo. L'ambiguïté découverte en handoff Phase 5c (calcul initial sur turbo = ~10× plus permissif via levier) a été fixée Phase 5c bis. Fallback turbo conservé déprécié (WARNING log) pour rétro-compatibilité tests legacy. **Anti-pattern à éviter** : confondre amplitude turbo et amplitude sous-jacent sur produits à levier — toujours préciser explicitement « sur sous-jacent » dans les specs et le code. | Tout futur produit avec instruments à frais fixes (turbos, options, futures à frais commission) | docs/ia/edge-scoring-model.md §3.2 SC2 v1.4, docs/product/functional-specs.md US-01, docs/analytics/edge-rnd-report.md §3 (filtre commun), src/scoring/sanity_checks.py SC2 (apply_sc2 + context underlying), src/backtester/edges/h_a_gap_follow.py et h_c_orb.py, tests/fixtures/ai/inputs/TC-0[1-8]-input.json (champs underlying_*) | ✅ corrigé (Phase 5c bis, 9 tests SC2 amplitude PASS dont 5 nouveaux underlying) | ✅ propagé (5 docs + 4 fichiers code + 9 tests + 8 fixtures) | À promouvoir en règle agent @data-analyst si réutilisé sur 2e projet à instruments à frais fixes : « calculer le breakeven net amplitude AVANT de définir le seuil de signal ; sur produits à levier, toujours expliciter sur QUEL prix l'amplitude se mesure (sous-jacent vs dérivé) ». Différent du R/R seul (qui mesure la qualité relative gain/risque sur prix dérivé) : l'amplitude mesure si le mouvement absolu absorbe les frais. Les deux complémentaires (SC2 v1.4 combine R/R turbo + amplitude sous-jacent). |
