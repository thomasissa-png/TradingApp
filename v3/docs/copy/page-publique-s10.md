## BLOC 1 — Nom de marque + Tagline

### Reco unique (à retenir)

**Nom : Issa Capital**
**Tagline : Le sens du marché. Mesuré.**

Justification : "Le sens" joue sur les deux sens du mot (direction + intuition affinée). "Mesuré" dit deux choses en un mot : on quantifie, et on reste sobre. Pas de promesse. Pas de verbe d'action qui survend. Calibré pour un primo-visiteur comme pour un trader confirmé.

### Alternatives (non recommandées, pour discussion uniquement)

- Alternative A : **Issa Capital · Lis le marché avant qu'il parle.**
  (plus narratif, légèrement plus mystérieux, fonctionne mieux à l'oral)

- Alternative B : **Issa Capital · La tendance, rien que la tendance.**
  (plus direct, plus radical, peut intimider un non-trader)

## BLOC 3 — Comment ça marche (3 étapes)

[Framework : FAB — chaque étape livre une Feature + Benefit implicite]

Titre de section (à afficher au-dessus des 3 étapes) :
```
Comment ça marche
```

### Étape 1
```
Le système analyse 12 marchés (or, pétrole, indices, devises, matières premières agricoles)
sur les données de prix et de momentum du moment.
```

### Étape 2
```
Pour chaque marché et chaque horizon (24h, 7 jours, 1 mois),
il tranche une direction : tendance haussière (LONG) ou baissière (SHORT).
```

### Étape 3
```
Le bulletin est publié 3 fois par jour. Vous lisez, vous décidez.
Le système ne place aucun ordre.
```

Version ultra-courte (si le design exige 1 ligne stricte par étape) :
```
1. Analyse les données de prix et de momentum en temps réel.
2. Tranche une direction pour chaque marché × chaque horizon.
3. Publie le bulletin 3 fois par jour. Vous restez aux commandes.
```

## BLOC 4 — Micro-explications (lexique intégré à la page)

Ces blocs s'affichent dans la section "Comment lire" ou en tooltip au survol des termes.

### LONG / SHORT — définition en 1 phrase
```
LONG : le système anticipe une tendance à la hausse sur cet actif et cet horizon.
SHORT : il anticipe une tendance à la baisse.
```

Version condensée pour tooltip ou badge :
```
LONG = hausse attendue · SHORT = baisse attendue
```

### Win rate — définition en 1 phrase
```
Le taux de bonnes directions : sur 100 fois où le bulletin a dit LONG ou SHORT,
combien de fois la tendance a effectivement suivi cette direction.
```

Version condensée :
```
Win rate : % de fois où la direction annoncée était la bonne.
```

### Horizons 24h / 7j / 1m — définition en 1 phrase
```
Chaque bulletin donne une direction sur 3 fenêtres de temps :
les prochaines 24 heures, les 7 prochains jours, et le prochain mois.
```

Version condensée pour en-tête de colonne :
```
24h : tendance court terme · 7j : tendance semaine · 1m : tendance mensuelle
```

## BLOC 5 — Statut "mode test / validation"

[Principe : honnêteté totale, tournée en force. "On mesure avant de promettre."]

### Version badge (court, dans le header)
```
En phase de mesure
```

Alternative plus explicite si la place le permet :
```
Phase test · Mise en service prévue 08/08
```

### Version 1 phrase (sous le hero ou bandeau discret)
```
Le système est en phase de mesure : les bulletins sont produits et enregistrés,
mais aucun signal n'est diffusé publiquement avant d'avoir validé le taux de justesse
sur une période réelle.
```

Version plus courte si l'espace est contraint :
```
En test depuis [date début] : on mesure la justesse des directions avant toute mise en service.
```

### Version bandeau (si on veut être encore plus explicite pour un primo-visiteur)
```
Ce que vous voyez ici est un enregistrement des bulletins produits en mode test.
Les directions ne sont pas diffusées comme signaux opérationnels.
Objectif de mise en service : 08/08/2026.
```
